# 阶段2：架构设计（架构师角色）

## 目标
基于 PRD 和现有代码库，产出完整的技术方案文档。核心产出除了架构设计本身，还包括一份「项目上下文摘要」供下游阶段（任务拆解、编码、QA）直接引用，确保全流水线对项目环境的理解一致。

---

## 执行步骤

### 2.1 加载输入

1. 用 Read 工具读取 `docs/<功能名>-prd.md`
2. 如果用户直接提供了 PRD 路径，从该路径读取

### 2.2 代码库分析

使用 Glob 和 Grep 工具探索现有代码库，产出「项目上下文摘要」：

**扫描清单：**

| 扫描项 | 目的 | 探测方法 |
|:-------|:-----|:---------|
| 技术栈 | 确定语言、框架、构建工具 | 查找 pom.xml / package.json / go.mod / requirements.txt / Cargo.toml 等 |
| 分层架构 | 理解模块划分 | 扫描目录结构（controller/service/dao/model 或 src/components/pages 等） |
| 相关代码 | 找到可复用的模块 | Grep 搜索 PRD 中的核心业务概念 |
| 数据库 | 理解数据模型 | 查找 migration/schema/DDL 文件，或扫描 entity/model 类 |
| 认证鉴权 | 理解安全机制 | 查找 auth/security/permission 相关代码 |
| 测试框架 | 确定测试方式 | 查找 test 目录、测试配置文件 |
| 构建与运行 | 确定命令 | 检查 Makefile / scripts / package.json scripts / pom.xml plugins |
| 配置管理 | 确定配置方式 | 查找 application.yml / .env / config 目录 / Apollo 配置等 |

**输出：项目上下文摘要**（嵌入到技术方案文档开头）

```markdown
## 项目上下文摘要

| 项 | 值 |
|:---|:---|
| 语言 | Java 17 |
| 框架 | Spring Boot 3.x + MyBatis-Plus |
| 前端 | Vue 3 + Element Plus（如有） |
| 构建命令 | mvn compile |
| 测试命令 | mvn test |
| 运行命令 | mvn spring-boot:run |
| 分层架构 | Controller → Service → Mapper → Entity |
| 配置方式 | Apollo 配置中心 |
| 认证方式 | JWT / IDaaS |
| 代码风格 | [观察到的命名约定、注释风格] |

### 可复用模块
- [已有模块名]: [可复用的原因和方式]

### 需新建模块
- [新模块名]: [职责说明]
```

这份摘要是后续所有阶段的共同参考。Phase 4 编码 Agent 会直接引用其中的技术栈、分层架构和代码风格约定。

### 2.3 架构设计

产出以下内容：

#### 2.3.1 系统架构总览
- 模块划分和职责
- 模块间依赖关系（用 Mermaid 图）
- 与现有系统的集成点

#### 2.3.2 API 设计
- RESTful API 列表（方法、路径、请求/响应结构）
- 认证和权限控制方案
- 错误码设计

#### 2.3.3 数据模型
- 新增/修改的数据库表结构（包含字段类型、索引、注释）
- 实体关系图（Mermaid ER 图）
- 与已有表的关联关系

#### 2.3.4 关键技术决策
- 技术选型及理由（复用已有技术栈，不引入新框架除非有充分理由）
- 性能优化策略
- 安全防护措施

#### 2.3.5 实现策略
- 推荐的实现顺序（对应 Phase 3 的波次）
- 风险点和应对方案
- 需要关注的边界条件

### 2.4 缺口检测

对照 PRD 逐项检查技术方案的完整性：

| 检查维度 | 说明 |
|:---------|:-----|
| PRD 功能覆盖 | 每个 PRD 场景是否有对应的技术实现 |
| API 完整性 | 每个用户操作是否有对应 API |
| 数据模型 | 每个业务实体是否有对应表/字段 |
| 异常处理 | PRD 中的异常场景是否有处理方案 |
| 非功能性 | 性能、安全、并发控制是否已设计 |

发现缺口时自动补充，无法确定时询问用户。

### 2.5 保存与审核

技术方案保存到 `docs/<功能名>-architecture.md`，文档开头必须包含「项目上下文摘要」。

用 AskUserQuestion 询问：
- "技术方案是否满足要求？"
  - 选项：
    - "确认通过，进入任务拆解阶段"
    - "需要修改"（附修改意见后迭代）
    - "暂停流水线"
